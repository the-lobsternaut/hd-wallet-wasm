Understanding Derivation Paths

BIP-32 defines a tree structure for deriving keys. Each level in the path represents a child key derived from its parent. The path notation uses forward slashes to separate levels, with an apostrophe (') indicating hardened derivation.

Level Name Description Typically Hardened?
m Master Root of the tree, derived from seed N/A
purpose' Purpose BIP number (44, 49, 84, etc.) Yes
coin_type' Coin Type SLIP-44 coin index (0=BTC, 60=ETH) Yes
account' Account Account index for user separation Yes
change Change 0=external (receive), 1=internal (change) No
index Address Index Sequential address index No
Hardened vs Non-Hardened Derivation

Hardened derivation (marked with ') prevents deriving parent private keys from child keys, even if the parent public key is known. The first three levels are typically hardened to prevent security issues if an xpub is compromised.
